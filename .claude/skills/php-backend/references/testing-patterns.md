# PHP Backend Testing Patterns (Laravel + PHPUnit / Pest)

## File layout

```
tests/
  TestCase.php               # base class — extends Laravel's Testing\TestCase
  CreatesApplication.php     # trait for bootstrapping the app
  Feature/                   # HTTP-level tests — boot the full app
    Auth/
      LoginTest.php
    Api/
      V1/
        UserControllerTest.php
  Unit/                      # isolated logic — no app boot needed
    Services/
      UserServiceTest.php
  Pest.php                   # Pest config (only if using Pest)
phpunit.xml                  # PHPUnit config
database/
  factories/
    UserFactory.php
```

## Naming & convention

- Class name ends in `Test` (e.g. `LoginTest.php` → `class LoginTest extends TestCase`)
- Test methods prefixed `test_` OR annotated with `#[Test]` attribute (PHP 8.1+) OR `/** @test */` docblock
- Run:
  - `php artisan test` — Laravel wrapper (adds nice output + env=testing)
  - `./vendor/bin/phpunit` — raw PHPUnit
  - `./vendor/bin/pest` — if using Pest
  - `php artisan test --filter=UserServiceTest` — single class
  - `php artisan test --coverage` — with coverage (needs Xdebug or PCOV)

## phpunit.xml — env overrides for tests

```xml
<php>
    <env name="APP_ENV" value="testing"/>
    <env name="DB_CONNECTION" value="sqlite"/>
    <env name="DB_DATABASE" value=":memory:"/>
    <env name="CACHE_STORE" value="array"/>
    <env name="QUEUE_CONNECTION" value="sync"/>
    <env name="SESSION_DRIVER" value="array"/>
    <env name="MAIL_MAILER" value="array"/>
</php>
```

`sqlite :memory:` is the fastest option for tests that need a real DB — migrations run per test class when combined with `RefreshDatabase`.

## Base TestCase (PHPUnit)

```php
namespace Tests;

use Illuminate\Foundation\Testing\TestCase as BaseTestCase;

abstract class TestCase extends BaseTestCase
{
    use CreatesApplication;
}
```

`CreatesApplication` is auto-generated by `laravel/laravel`; it bootstraps the container.

## Feature test — HTTP endpoint

```php
namespace Tests\Feature\Api\V1;

use App\Models\User;
use Illuminate\Foundation\Testing\RefreshDatabase;
use Tests\TestCase;

class UserControllerTest extends TestCase
{
    use RefreshDatabase; // wipes + re-migrates per test

    public function test_guest_cannot_list_users(): void
    {
        $this->getJson('/api/v1/users')
             ->assertStatus(401);
    }

    public function test_authenticated_user_can_create(): void
    {
        $admin = User::factory()->admin()->create();

        $response = $this->actingAs($admin, 'sanctum')
            ->postJson('/api/v1/users', [
                'email'    => 'new@example.com',
                'password' => 'secret12',
            ]);

        $response->assertCreated()
                 ->assertJsonPath('data.email', 'new@example.com');

        $this->assertDatabaseHas('users', ['email' => 'new@example.com']);
    }

    public function test_validation_fails_on_duplicate_email(): void
    {
        User::factory()->create(['email' => 'dup@example.com']);
        $admin = User::factory()->admin()->create();

        $this->actingAs($admin, 'sanctum')
            ->postJson('/api/v1/users', [
                'email'    => 'dup@example.com',
                'password' => 'secret12',
            ])
            ->assertStatus(422)
            ->assertJsonValidationErrors(['email']);
    }
}
```

Key HTTP helpers: `getJson`, `postJson`, `putJson`, `patchJson`, `deleteJson` — they add `Accept: application/json` + serialize the payload. Assertions: `assertOk`, `assertCreated`, `assertNoContent`, `assertStatus($n)`, `assertJsonPath($path, $value)`, `assertJsonFragment([...])`, `assertJsonStructure([...])`, `assertJsonValidationErrors([...])`, `assertUnauthorized`, `assertForbidden`.

## Factories

```php
namespace Database\Factories;

use App\Models\User;
use Illuminate\Database\Eloquent\Factories\Factory;
use Illuminate\Support\Facades\Hash;

class UserFactory extends Factory
{
    protected $model = User::class;

    public function definition(): array
    {
        return [
            'email'    => $this->faker->unique()->safeEmail(),
            'password' => Hash::make('password'),
            'role'     => 'user',
        ];
    }

    public function admin(): static
    {
        return $this->state(fn () => ['role' => 'admin']);
    }

    public function unverified(): static
    {
        return $this->state(fn () => ['email_verified_at' => null]);
    }
}
```

Usage: `User::factory()->create()`, `User::factory()->admin()->create()`, `User::factory()->count(10)->create()`, `User::factory()->has(Order::factory()->count(3))->create()`.

## Unit test — service with mocked dependencies

```php
namespace Tests\Unit\Services;

use App\Models\User;
use App\Services\UserService;
use App\Repositories\UserRepository;
use Tests\TestCase;
use Mockery;

class UserServiceTest extends TestCase
{
    public function test_create_hashes_password(): void
    {
        $repo = Mockery::mock(UserRepository::class);
        $repo->shouldReceive('create')
             ->once()
             ->with(Mockery::on(fn ($data) =>
                 $data['email'] === 'a@b.com' && $data['password'] !== 'plain'
             ))
             ->andReturn(new User(['email' => 'a@b.com']));

        $service = new UserService($repo);
        $user    = $service->create(['email' => 'a@b.com', 'password' => 'plain']);

        $this->assertSame('a@b.com', $user->email);
    }

    protected function tearDown(): void
    {
        Mockery::close();
        parent::tearDown();
    }
}
```

Laravel ships with `MockeryPHPUnitIntegration` trait that auto-calls `Mockery::close()` if you `use` it.

## Authentication — `actingAs` with Sanctum

```php
// Token guard — for API routes under auth:sanctum
$this->actingAs(User::factory()->create(), 'sanctum')
     ->getJson('/api/v1/me')
     ->assertOk();

// Issue a real token (rare — only if you're testing token creation flow)
$user  = User::factory()->create();
$token = $user->createToken('test')->plainTextToken;
$this->withHeader('Authorization', "Bearer {$token}")
     ->getJson('/api/v1/me')
     ->assertOk();
```

## Queue / Job tests

```php
use Illuminate\Support\Facades\Queue;
use App\Jobs\SendWelcomeEmail;

public function test_registration_dispatches_welcome_job(): void
{
    Queue::fake();

    $this->postJson('/api/v1/auth/register', [
        'email'    => 'new@example.com',
        'password' => 'secret12',
    ])->assertCreated();

    Queue::assertPushed(SendWelcomeEmail::class, fn ($job) =>
        $job->userId === User::where('email', 'new@example.com')->first()->id
    );
}
```

Other fakes: `Mail::fake()` + `Mail::assertSent(WelcomeMail::class)`, `Event::fake()` + `Event::assertDispatched(UserRegistered::class)`, `Notification::fake()`, `Bus::fake()`, `Storage::fake('public')`.

## Cache / Redis tests

Tests use `CACHE_STORE=array` by default (see `phpunit.xml`) — no Redis needed. To test Redis-specific behavior:

```php
use Illuminate\Support\Facades\Cache;

public function test_user_is_cached(): void
{
    $user = User::factory()->create();
    $this->getJson("/api/v1/users/{$user->id}")->assertOk();
    $this->assertTrue(Cache::has("user:{$user->id}"));
}

public function test_cache_invalidated_on_update(): void
{
    $user = User::factory()->create();
    Cache::put("user:{$user->id}", $user, 600);

    $admin = User::factory()->admin()->create();
    $this->actingAs($admin, 'sanctum')
         ->patchJson("/api/v1/users/{$user->id}", ['role' => 'admin']);

    $this->assertFalse(Cache::has("user:{$user->id}"));
}
```

## Pest — modern alternative

```php
// tests/Feature/Api/V1/UserControllerTest.php
use App\Models\User;
use function Pest\Laravel\{actingAs, postJson, assertDatabaseHas};

uses(\Illuminate\Foundation\Testing\RefreshDatabase::class);

it('creates a user', function () {
    $admin = User::factory()->admin()->create();

    actingAs($admin, 'sanctum')
        ->postJson('/api/v1/users', [
            'email'    => 'new@example.com',
            'password' => 'secret12',
        ])
        ->assertCreated();

    assertDatabaseHas('users', ['email' => 'new@example.com']);
});

it('rejects duplicate email', function () {
    User::factory()->create(['email' => 'dup@example.com']);
    $admin = User::factory()->admin()->create();

    actingAs($admin, 'sanctum')
        ->postJson('/api/v1/users', [
            'email'    => 'dup@example.com',
            'password' => 'secret12',
        ])
        ->assertStatus(422);
})->group('validation');
```

Pest's advantages: terser syntax, datasets (`->with([...])`), higher-order tests, architecture tests (`arch()->expect('App\Models')->toExtend(Model::class)`). Install: `composer require pestphp/pest --dev --with-all-dependencies` + `./vendor/bin/pest --init`.

## Common pitfalls

- Forgetting `use RefreshDatabase` → tests bleed state, random failures on re-run
- `env('FOO')` in code → cached in prod, returns null in tests. Use `config('foo.bar')` and set via `config(['foo.bar' => 'x'])` inside tests
- Not faking `Queue` / `Mail` / `Event` → real dispatch happens, slow + flaky
- Testing Eloquent internals instead of behavior → brittle. Prefer `assertDatabaseHas` over `$mock->shouldReceive('save')`
- Running feature tests without `$this->withoutExceptionHandling()` when debugging 500s → you see only the sanitized response, not the real stack
- Hardcoding IDs (`->create(['id' => 1])`) → breaks when test order changes
- Shared `static` state in services → carries over between tests; reset in `setUp`
- Factory using real API calls (Faker `imageUrl()` hits fake Lorem Picsum) → offline CI breaks. Use seeded faker or fixture URLs
- `actingAs($user)` without guard name when using Sanctum → hits default web guard, token middleware rejects anyway. Always pass `'sanctum'`

## TDD with Laravel

Red phase checklist:
1. Write `UserControllerTest::test_can_create_user` — hits `POST /api/v1/users`
2. Route doesn't exist → 404 → test fails ✅
3. Add route → controller missing → 500 → still fails ✅
4. Add controller with `abort(501)` → test fails on assertion ✅
5. Only now implement `UserService::create` → test passes (Green)
6. Refactor: extract FormRequest, add Resource, add cache invalidation (Green stays)

Run single test during red-phase: `php artisan test --filter=test_can_create_user --stop-on-failure`.
